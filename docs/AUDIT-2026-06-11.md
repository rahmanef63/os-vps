## 1. P0 — broken now, user-visible

1. **image-editor: Export/Save renders the viewport, not the document** —
   `frontend/slices/image-editor/lib/export.ts:18` calls
   `stage.toDataURL({...})` with no x/y/width/height: output includes
   pasteboard padding, zoom-dependent resolution, **Transformer handles and the
   drop-shadow baked into the file**, black surround on JPEG. Affects Export
   panel, ⌘Save, mobile top-bar, AI `export.image`, and the close-guard save.
   Fix: pass the doc rect + `pixelRatio: scale / zoom`, hide the overlay layer
   during render.
2. **image-editor: failed project load auto-clobbers the saved file** —
   `image-editor.tsx:117-127` — `fetch(src)…catch(() => {}).finally(onDone)`
   flips `ready=true` even on failure; 800 ms later the autosaver writes the
   blank starter doc **over the user's file**. One network blip = file
   destroyed. Fix: only `onDone()` on success; error toast otherwise.
3. **Settings: "Laptop" SSH target is uneditable** —
   `lib/appearance/server-targets.ts:37` — `uniqueTargets([...legacyVps,
   ...existing])` puts pristine `CORE_TARGETS` *first* in a first-wins dedupe
   and `ensureServerTargets` runs on every `setServer`, so each keystroke in
   host/user/port is instantly reverted. Fix: invert precedence (saved first),
   append only *missing* core ids.
4. **code-editor: ⌘S while typing opens the browser Save dialog** —
   `components/editor-surface.tsx:98` — the textarea's `onKeyDown` does
   `e.stopPropagation()` on every key, so the window-level save listener
   (`app.tsx:47-56`, bubble phase) never sees ⌘S in the only state where you'd
   press it. Fix: handle ⌘S in the textarea handler (or use capture phase).
5. **files-manager: context-menu "Download" is a stub** — `app.tsx:183`
   `onDownload={() => fs.setError(null)}`. Fix: anchor to
   `rawUrl(joinPath(fs.path, name))` with the `download` attr (media-viewer has
   the pattern), or remove the item.

## 2. P1 — break under realistic conditions

### Shell / window manager (verified 3× independently where noted)

- **Resize commits viewport coords into section-relative state** —
  `appshell/hooks/use-window-drag.ts:96-99` uses `getBoundingClientRect()`
  where the drag path 40 lines above documents why `offsetLeft/offsetTop` is
  required. On macOS shell (`top-[30px]` section) every resize drops the
  window +30px, accumulating. Two-line fix. *(Found by 3 independent passes.)*
- **Spotlight self-sustains an infinite search loop** —
  `os-shell/capabilities.ts:42-52` returns a fresh async closure per render;
  `shell-search/components/spotlight.tsx` keys its effect on that identity →
  `/api/v1/fs/search` fires every ~150 ms while the palette is open with text.
  The capability contract (`registry/capabilities.tsx:79-83`) explicitly
  forbids this. Fix: `useCallback` on `api`. *(Re-verified by hand.)*
- **Deep-link reload duplicates a Files window every refresh** —
  `appshell/lib/store-persist.ts:59-64` — `hydrateBoot` drops a persisted
  window only for single-instance apps; for `multiApps` the UrlSync-spawned
  live window and the persisted copy of the *same* window both survive,
  +1 per reload, unbounded. Fix: also drop a persisted multi-app window whose
  payload deep-equals a live one. *(Re-verified by hand.)*
- **No re-clamp of restored/free windows** — `store-persist.ts:31-70` restores
  raw rects; nothing listens to viewport `resize`; `retileSnapped()` only runs
  from the Windows-shell chrome insets. Shrink the browser → title bars land
  offscreen, unreachable. Fix: clamp in `hydrate*` + debounced resize handler.
- **Visible stacking ignores `win.z`** — `components/window.tsx:57` gives all
  unfocused windows `z-10`; DOM order = creation order. Focus an old window,
  click another → it sinks to creation depth, ⌘Tab "MRU" is actually
  creation-order, `closeWindow` focuses by insertion. Fix: render sorted by
  `z` (or move-to-end of `order` on focus).

### Data integrity (apps)

- **files-manager rename-to-existing silently destroys the target**
  (`hooks/use-file-commands.ts:94` — POSIX rename clobbers; the `taken` set is
  already available). Also: **cut+paste into the same folder renames files in
  place** (`use-file-ops.ts:107-108`), and **failed listing renders as an empty
  folder** with a "Drop files to upload" invite (`use-files.ts:51-53`).
- **image-editor paint-pixel loss cluster** (`lib/doc-ops.ts:22-72`): delete
  layer kills its canvas before the history push (undo restores blank);
  duplicate never copies pixels; crop re-bakes destructively; doc resize
  never resizes paint canvases. Plus **`blob:` URLs serialized into autosave**
  (`menu-bar.tsx:52` → dead after reload, layers silently vanish) and
  **window-global hotkeys** (Backspace deletes a layer while another window is
  focused, `hooks/use-keyboard.ts:38-46`).
- **media-studio doc save is fire-and-forget and bypasses the os-api seam** —
  `app.tsx:95-100` `void fetch("/api/v1/fs/write")`: no `.ok`, no catch, hits
  live routes even in mock/demo. Close-prompt "Save" force-closes (discarding
  work) when export returns null (`app.tsx:69`).
- **code-editor stale-buffer cluster** (`lib/use-editor.ts:36,57,102`):
  reopened tab shows stale content then Save overwrites a newer host file;
  payload-open clobbers in-flight edits; optimistic `setDisk` makes a failed
  save unretryable; no close guard for dirty tabs; the Inspector "Save" action
  runs a stale closure (`app.tsx:74-80`).
- **prefs read-modify-write race** — `lib/prefs/store.ts:38`: two debounced
  section POSTs within ~1.5 s interleave; one section silently lost.

### Streams / resources

- **assistant: no abort seam end-to-end** — `chat-panel.tsx:73`,
  `lib/ai/stream.ts:63-68`, `app/api/assistant/route.ts:82-105`: closing the
  window keeps billing Anthropic tokens to completion; mid-stream error
  **replaces** the partial reply; switching in-app tabs unmounts and destroys
  the conversation. Fix: wire `req.signal` + `AbortController`, append errors,
  lift messages out of component state.
- **browser: window close leaks remote Chromium pages** —
  `lib/use-remote-browser.ts:139-144` never calls `api.close` on unmount and
  the service reaper defaults off (`os-browser/server.mjs:25`, `IDLE_MS=0`).
  Also: service-down = eternal "Loading remote browser…" (`lib/host.ts:49-55`
  swallows errors, polls a dead endpoint forever); RemoteView is
  functionally desktop-only (no touch scroll/keyboard/zoom).
- **reel-editor: render outlives window close** (`lib/use-export.ts` — records
  black frames against a disposed MediaCache, then announces success);
  same-source audio clips share one GainNode (`lib/media-audio.ts:43,98` — one
  copy's envelope mutes the other); imported object URLs never revoked.
- **system-monitor: stats failure = infinite spinner** +1 unhandled rejection
  per 1.5 s (`lib/use-stats-history.ts:33-35`); polling never pauses hidden.

### Error reporting / misc

- **`lib/os-api` discards the server's JSON error body** —
  `http-adapter.ts:27` throws `\`${status} ${statusText}\`` and statusText is
  empty on HTTP/2 → every live-mode error in the OS reads `"403 "` while the
  route's descriptive `{error}` is thrown away. **One fix improves every error
  message in the product.** Also: `statsStream` and `auth.*` adapter methods
  target routes that don't exist (mock masks it).
- **quicklinks hydrate trusts parsed shape** — `lib/quicklinks/store.tsx:74`
  (no `Array.isArray`): one bad localStorage row crashes dock + Launchpad +
  widget + Settings at once (the `applyServer` path has the guard).
- **login: errors invisible in the pending-device state**
  (`auth/components/login-screen.tsx:118` — error `<p>` only renders in the
  `!pending` branch); unknown statuses all masquerade as "Password salah."
- **app-store: touch users uninstall by tapping "Installed"** —
  hover-revealed destructive swap, no hover on coarse pointers, no confirm
  (`store-app-card.tsx:65-74`, `system-card.tsx:64-73`).
- **create-app: empty name persisted** while the preview shows a fallback
  (`app.tsx:96`); custom slug can collide with built-in app ids.
- **image-picker: link "verification" is theater** — `LinkTab.tsx:20-25`
  `img.onerror = () => resolve()` then selects unconditionally.
- **Phone-frame escape**: Control Center uses a Radix Sheet portal to
  `document.body` (fixed) so in framed mobile preview it covers the whole
  monitor (`control-center.tsx:33-36`); toasts likewise.
- **mobile-switcher mounts live `WindowContent` per card** —
  `mobile-switcher.tsx:129-133`: a second live instance of every open app
  (double PTY/screencast sessions) while the switcher is open, and no `winId`
  → close guards dead on mobile. Fix: static snapshot cards (desktop
  `WindowOverview` already does this).

## 3. Security (no P0/P1 — hardening list)

Verified solid: every `/api/*` route gates with `verifyAuth`/`requireSession`
first (coverage table checked route-by-route); realpath-before-bounds jail
with no working traversal/symlink escape (tmp+rename writes never write
*through* a symlink); timing-safe compares with fail-closed weak-secret
rejection; `SameSite=Strict` + `Sec-Fetch-Site` CSRF depth-2; `~/.os-vps`
0700/0600 atomic writes; BYOK key masked; SVG served with `CSP: sandbox`;
demo mode can't be flipped into an open host shell.

- **P2 — `OS_AGENT_TOKEN` is documented as browser-only but unlocks the entire
  host API** (`lib/agent/server.ts:81-84` — `verifyAuth` accepts it for exec /
  fs / term too; no expiry, no rotation, bypasses device approval). Either
  scope it to `/api/v1/browser/*` or rewrite `.env.example` to say "full host
  control". Also attribute its audit lines (currently `actor: null`).
- **P2 — sensitive-file denylist incomplete vs. its marketing**
  (`lib/host/paths.ts:62`): blocks `.ssh`/`.bash_history` but not `.aws`,
  `.kube`, `.docker`, `.config/gcloud`, `.netrc`, `.git-credentials`,
  `.zsh_history`. Extend or soften the README claim.
- **P2 — browser fill/type audit lines persist typed text** —
  `lib/agent/server.ts:52-54`: passwords typed into third-party sites via the
  Browser app land in plaintext in `~/.os-vps/audit.log`. Redact value/text.
- **P2 — child processes inherit the full secret env** (`lib/host/exec.ts:89`,
  `pty.ts:90`): one `printenv` reveals `OS_SESSION_SECRET` etc. — defeats the
  denylist's stated purpose (persistence after revocation). Scrub `OS_*` +
  `ANTHROPIC_API_KEY` from child env.
- **P3** — login length-leak via the `a.length === b.length` short-circuit
  (hash both sides to fixed width first, as agent-token already does); no HSTS
  at app layer (set at the TLS proxy and document); device factor is a static
  JS-readable bearer id, not a challenge (acceptable single-owner, document
  it); `/api/assistant` has no rate limit (owner's own spend).
- **Doc fix in your favor**: README claims revoking a device doesn't kill live
  sessions — it does (`getSession` re-checks `isApproved` per request).

## 4. UX — cross-cutting patterns (the systemic stuff)

1. **Seven error-surface regimes.** Terminal = inline red lines; Files =
   inline bar for mutations but *swallowed* for listings + toasts for uploads;
   browser = swallowed or success-styled; monitor = spinner-only; code-editor
   = status-bar text; assistant = injected into the reply bubble; Settings =
   toast for a passive read, silence for actionable failures. The shell toast
   bus exists and almost no app uses it. **Pick one doctrine** (inline for
   in-flow validation, toast bus for async/background, full-state for
   listings) and sweep.
2. **Destructive ops mostly unconfirmed.** Trash-first exists only in Files
   (no Restore). Zero confirm for: layer delete (one keystroke), agent/skill
   delete, app uninstall (hover-mislabeled on touch), quicklink remove, device
   revoke, "New project" wipe, dirty-tab close. media-studio's close guard
   (`setCloseGuard` veto + Save/Don't Save/Cancel) is the model — adopt it in
   code-editor and reel-editor first, where work loss hurts most.
   `ResponsiveDialog` was built for this and has zero adopters; three apps
   still call `window.confirm`.
3. **Shortcut chaos in both directions.** Advertised-but-unbound: browser ⌘T/
   ⌘R (do the *host* browser action), image-editor ⌘S/⇧⌘S/⌘W/Enter-to-crop,
   reel ⌘E/⌘D/⌘±. Inverse: window-global listeners firing when the app isn't
   focused (code-editor ⌘S, image-editor Backspace, Space swallowed OS-wide).
   The shell needs a **focused-window-scoped hotkey capability**; every app is
   reinventing it wrong. Also: `Ctrl+Shift+V` (clipboard overlay) and `⌘I`
   (inspector) fire inside inputs/terminal — both need the `inEditable()`
   guard that quick-look already uses.
4. **i18n**: exactly six Indonesian strings in an otherwise English product
   (4 login errors, `"Gagal membuat app"`, `"App slug sudah dipakai"`). Pick a
   locale per surface.
5. **A11y**: hand-rolled menus/tabs without roles or keyboard support
   (files context menu, image-picker tabs, browser tabs, assistant tabs);
   icon buttons unlabeled in media-viewer toolbar; `maximumScale: 1`
   in `app/layout.tsx:20-21` (WCAG 1.4.4); Files list-view keyboard-dead.
6. **Stale slice.json metadata misleads** — 8 slices still declare Convex deps
   that don't exist (`auth/slice.json` describes a JWT/JWKS auth that isn't
   the real HMAC-cookie one); 3 contracts say `auth: "convex"`.

## 5. Performance

- **Pollers without visibility gating**: system-monitor 1.5 s ×2 requests,
  capabilities CPU 4 s + sys/disk 3 s (one interval per mounted consumer —
  menu-bar chip + Today widget poll concurrently), browser MJPEG screencast
  never pauses (only the JPEG fallback checks `document.hidden`). Gate all on
  `visibilitychange`; share one poller per capability.
- **Re-render hotspots**: every screencast frame re-renders the whole browser
  chrome (`shot` is top-level state); reel playback `setFrame` re-renders the
  entire slice per frame (zero `React.memo`); code-editor re-tokenizes the
  full document per keystroke; assistant per-token full-array map +
  forced `scrollIntoView`.
- **Store-subscription tearing class**: `dock.tsx:37` and
  `menu-bar-menus.tsx:126` read `shellStore.getWindow()` during render while
  subscribed only to `order`/`focused` → stale dock hover lists / Window-menu
  marks on `patch()`-only updates. Use per-window `useWindow(id)` (the
  taskbar already does it right). Related: `registry.tsx:16,23` unmemoized
  Map + fresh array defeats every `useMemo(..., [apps])` downstream;
  `clipboard.ts:105`/`recents.ts:64` return mutated module state as
  `getServerSnapshot` (copy `shells.tsx:131`'s constant pattern).
- **Memory**: image-editor history holds up to 120 full-canvas data URLs;
  object-URL leaks in image-editor/reel/image-picker; reel MediaCache never
  evicts removed clips; Files streams full-resolution bytes as thumbnails, no
  virtualization (the one list that needs it).
- **Deps**: `tailwind-merge` 2.6 against Tailwind 4 (v2 targets v3 class
  tables — `cn()` conflict resolution can silently be wrong; bump to ^3);
  dual Radix import surface (5 files monolith vs 7 scoped) misaligned with
  `optimizePackageImports`.

## 8. What's genuinely good (keep doing this)

- The auth/jail core is among the cleanest implementations of this stack:
  per-request device revocation, fail-closed weak-secret rejection, SSE
  term stream with Last-Event-ID resume + heartbeat + abort cleanup.
- External-store discipline: 15+ pub/sub buses, all with stable snapshots,
  SSR guards, and leak-free unsubscribes.
- files-manager's stale-listing fix *by derivation* (`listing.key ===
  loadKey`) and the PTY lifecycle (serialized batches, keepalive close,
  8-session cap + reaper) are the reference patterns for the rest.
- Dock magnification math (constant-width gaussian pool, rAF outside React)
  and the capability/slot seam (defaults merged, accessors unconditional) are
  framework-grade.
- UnsplashTab's debounce + AbortController search is the best async snippet
  in the repo — copy it wherever fetch-on-type exists.
